🎯 USER INTERACTIONS
═════════════════════════════════════════════════════════════════════════════

1. Mouse Hover:
   • Dots within proximity change color
   • Gradient from baseColor to activeColor
   • Real-time tracking

2. Mouse Movement:
   • Fast movement (>speedTrigger) pushes dots
   • Stronger movement = stronger push
   • Inertia physics applied

3. Click:
   • Dots explode outward
   • Within shockRadius from click point
   • Falloff effect (stronger at center)

4. Return:
   • Elastic animation back to origin
   • Smooth bounce effect
   • Takes returnDuration seconds

🔧 PROP REFERENCE
═════════════════════════════════════════════════════════════════════════════

dotSize: number (default: 16)
  → Size of dots in pixels (8-24 recommended)

gap: number (default: 32)
  → Space between dots in pixels (20-50 recommended)

baseColor: string (default: #5227FF)
  → Default color in hex format

activeColor: string (default: #5227FF)
  → Color when mouse is near (hex format)

proximity: number (default: 150)
  → Distance to detect mouse in pixels (80-250)

speedTrigger: number (default: 100)
  → Min mouse speed to push dots (25-250 px/s)

shockRadius: number (default: 250)
  → Click explosion radius in pixels (150-400)

shockStrength: number (default: 5)
  → Push force multiplier (1-10)

maxSpeed: number (default: 5000)
  → Max tracked mouse speed (px/s)

resistance: number (default: 750)
  → Physics friction/resistance (500-1000)

returnDuration: number (default: 1.5)
  → Return animation time in seconds (0.5-3)

className: string
  → Additional CSS classes

style: CSSProperties
  → Inline styles

🐛 TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

Grid not showing?
  → Check z-index: grid needs lower z-index than content
  → Ensure parent has width/height defined
  → Check overflow is not hidden inappropriately

Not interactive?
  → Verify dots aren't covered by other elements
  → Check z-index of content above
  → Ensure mouse events aren't prevented

Performance issues?
  → Increase gap spacing (fewer dots)
  → Reduce dotSize
  → Increase proximity range or speedTrigger
  → Consider using on hero section only

Colors not showing?
  → Verify hex color format (#RRGGBB)
  → Check activeColor is different from baseColor
  → Try predefined preset colors

Mobile lag?
  → Use mobile preset (smaller dots, larger gap)
  → Reduce proximity range
  → Increase speedTrigger threshold
  → Test on actual device
